## Features

- **Priority Fee Configuration**: Choose between Normal, Medium, and Turbo priority levels
- **Priority Fee Cap**: Set dynamic or manual priority fee caps
- **Slippage Management**: Configure dynamic or fixed slippage tolerance
- **Custom Slippage Values**: Set specific slippage percentages (0.5%, 1%, 1.5%, or custom)
- **Context Provider**: Manage settings state across your application
- **Customizable Trigger**: Use any component as the settings trigger
- **Form Validation**: Built-in validation for numeric inputs
- **Responsive Design**: Works seamlessly on desktop and mobile

### useTxnSettings Hook

A hook to access and modify transaction settings from any component within the provider.

```tsx
const { settings, updateSettings, isOpen, setIsOpen } = useTxnSettings();
```

**Returns:**

- `settings`: Current transaction settings object
- `updateSettings`: Function to update settings
- `isOpen`: Dialog open state
- `setIsOpen`: Function to control dialog visibility

### TxnSettingsType

The transaction settings configuration object.

```tsx
type TxnSettingsType = {
  priority: string; // "normal" | "medium" | "turbo"
  priorityFeeCap: "dynamic" | number;
  slippageMode: "dynamic" | "fixed";
  slippageValue: number; // Percentage value
};
```

## Settings Configuration

### Priority Levels

- **Normal**: Standard transaction priority
- **Medium**: Increased priority for faster confirmation
- **Turbo**: Highest priority for immediate processing

### Priority Fee Cap

- **Dynamic**: Automatically adjusts based on network conditions
- **Manual**: Set a specific maximum fee amount

### Slippage Modes

- **Dynamic**: Automatically calculates optimal slippage
- **Fixed**: Use predefined or custom slippage values
  - Low: 0.5%
  - Normal: 1.0%
  - High: 1.5%
  - Custom: User-defined percentage

## Usage with Context

```tsx
import { useTxnSettings } from "@/components/soldevkit-ui/transaction/txn-settings";

function MyComponent() {
  const { settings, updateSettings } = useTxnSettings();

  const handlePriorityChange = () => {
    updateSettings({ priority: "turbo" });
  };

  return (
    <div>
      <p>Current priority: {settings.priority}</p>
      <button onClick={handlePriorityChange}>Set Turbo</button>
    </div>
  );
}
```
